### 1. ROSOrin Pro — Intégration Robotique OpenClaw
**Score: 8/10** 🟢
- **Source:** Let's Data Science (31 mars 2026)
- **Quoi:** HiWonder lance le ROSOrin Pro, plateforme robotique "OpenClaw-ready" connectant des LLMs à des actuateurs physiques via ROS 2 Humble + edge compute (Jetson Orin Nano / RPi 5).

### 2. Lobster — Workflow Shell natif OpenClaw
**Score: 7/10** 🟢
- **Source:** GitHub openclaw org (31 mars 2026)
- **Quoi:** Nouveau repo "Lobster" — un shell de workflow natif OpenClaw. Moteur de macros typé, local-first, qui compose les skills/tools en pipelines automatisés.
- **Pourquoi c'est intéressant:** Passe OpenClaw de "agent conversationnel" à "plateforme d'orchestration". Les skills deviennent composables comme des commandes Unix.

### 3. NanoClaw — Alternative légère conteneurisée
**Score: 6/10** 🟡
- **Source:** GitHub qwibitai/nanoclaw (31 mars 2026)
- **Quoi:** Alternative lightweight à OpenClaw tournant dans des conteneurs Docker, basée sur Anthropic's Agents SDK. Supporte WhatsApp, Telegram, Slack, Discord, Gmail.
- **Pourquoi c'est intéressant:** Adresse le problème de sécurité #1 d'OpenClaw (accès filesystem complet). L'isolation par conteneur est un vrai argument pour l'adoption enterprise.
- **Potentiel:** Pourrait fragmenter la communauté ou pousser OpenClaw à améliorer sa sécurité.
