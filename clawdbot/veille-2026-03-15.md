## 📝 Analyse "92.5% des use cases n'ont pas besoin d'OpenClaw" (Score: 7/10)

Article viral sur DEV.to par @miss_terry. Analyse du repo `awesome-openclaw-usecases` (40 use cases) :
- **50%** sont agent-agnostic (marchent avec n'importe quel agent)
- **42.5%** partiels (un outil à changer)
- **Seulement 7.5%** (3/40) nécessitent vraiment OpenClaw

**Insight clé:** L'auteur a créé un "Index Skill Pattern" — SKILL.md comme table des matières (~1200 tokens) avec chargement à la demande (~800 tokens/use case). 93% d'économie de contexte vs monolithique.

**Use cases catalogués:** Second brain, self-healing server, podcast pipeline, CRM, habit tracker, arXiv reader, Polymarket autopilot, etc.

**Repo:** `devjiro76/not-need-claw` — installable via `npx skills add`

**→ Pertinence pour nous:** Le pattern "Index Skill" est intelligent pour nos propres skills dotclaud. À considérer.

## 💼 Use Cases Business Avancés (Score: 7/10)

Article RoboRhythms — retour d'expérience des power users :

1. **Pipeline prospection LinkedIn** — Skill BeReach : scrape profils, scoring ICP, drafts messages. Pipeline overnight, review 20 min le matin
2. **Reports clients automatisés** — Pull GA + CRM + spreadsheet → rapport hebdo formaté → email auto chaque vendredi
3. **Monitoring concurrentiel** — Agents qui surveillent pricing, pages produit, offres d'emploi des concurrents
4. **Extraction factures/contrats** — OCR PDF → routing données structurées

**Pattern dominant chez les power users:**
- **VPS-Telegram stack** — $6/mois DigitalOcean, agent H24, contrôle depuis le téléphone
- **Agent specialization** — Agents étroits chaînés (extraction → scoring → draft)

**→ Pertinence pour nous:** Le pipeline factures est exactement notre projet invoice-auditor. Le monitoring concurrentiel est applicable à tous nos SaaS.
