### OpenAI lance GPT-5.5-Cyber + initiative "Patch the Planet"
- **Quoi** : Modèle spécialisé en cybersécurité avec détection de vulnérabilités et génération de patchs automatisée ; initiative open-source co-fondée avec Trail of Bits et HackerOne
- **Lien** : https://openai.com/index/patch-the-planet/
- **Pourquoi c'est important** : GPT-5.5-Cyber atteint 85,6 % sur CyberGym (vs 81,8 % pour GPT-5.5 standard) et 39,5 % sur ExploitGym. Plus de 30 projets open-source participants : cURL, Go, Python, Sigstore… Le pari : l'IA a rendu la détection de bugs facile, le nouveau goulot d'étranglement est le patching.
- **Testable** : Oui — via API OpenAI (accès Codex Security + modèle GPT-5.5-Cyber)

### Claude Fable 5 n'est plus gratuit — fin de la période d'essai

## 🆕 Nouveaux Outils & Produits

### Vercel Eve — framework open-source pour agents IA
- **Quoi** : Framework TypeScript-natif open-source (Apache 2.0) où chaque agent IA est un dossier de fichiers qui compile vers un service Vercel Functions
- **Lien** : https://vercel.com/blog/introducing-eve
- **Pourquoi c'est important** : "Next.js for agents" selon Vercel. Inclut exécution durable, compute sandboxé, approvals human-in-the-loop, sous-agents, tracing OpenTelemetry et système d'evals — tout ça par défaut. Déploiement via `vercel deploy` comme une app web classique.
- **Testable** : Oui — open-source sur GitHub, Apache 2.0, déploiement Vercel gratuit pour débuter

### Mistral Le Chat rebaptisé Vibe
- **Quoi** : Mistral renomme Le Chat en "Vibe" et lance 3 modes : Vibe Work (agent productivité multi-apps), Vibe Code (CLI + extension VS Code + web), Vibe Chat (conserve l'expérience conversationnelle)
- **Lien** : https://help.mistral.ai/en/articles/682992-le-chat-is-now-vibe
- **Pourquoi c'est important** : Vibe Work s'intègre à Google Workspace, Outlook, SharePoint, Slack et GitHub. Vibe Code rivalise directement avec Claude Code et Cursor. Même URL (chat.mistral.ai), même login.
- **Testable** : Oui — chat.mistral.ai, gratuit pour commencer

### Microsoft Scout — agent autonome d'entreprise
- **Quoi** : Agent "always-on" avec identité Entra propre, propulsé par OpenClaw, accessible via les Work IQ APIs (MCP tools + semantic index)
- **Lien** : https://releasebot.io/updates/openai
- **Pourquoi c'est important** : Premier agent Microsoft avec une identité gouvernée (Entra), permettant audit et contrôle d'accès enterprise. Les Work IQ APIs ouvrent l'indexation sémantique de l'organisation à des agents tiers via MCP.
- **Testable** : Non — preview enterprise

### xAI Grok 4.3 sur Amazon Bedrock (GA)
- **Quoi** : Grok 4.3 disponible en général sur AWS Bedrock, contexte 1M tokens, niveaux de raisonnement configurables
- **Lien** : https://releasebot.io/updates/xai
- **Pourquoi c'est important** : Permet d'intégrer Grok dans des workflows AWS existants. Low hallucination revendiqué pour enterprise. xAI positionne Grok comme alternative sérieuse à Claude et GPT dans les pipelines cloud.
- **Testable** : Oui — Amazon Bedrock console, usage-based pricing

### Google Gemini Spark — agent personnel cloud
- **Quoi** : Agent 24/7 annoncé à Google I/O, qui tourne en arrière-plan même quand le téléphone est verrouillé
- **Lien** : https://techcrunch.com/2026/05/19/google-updates-its-gemini-app-to-take-on-chatgpt-and-claude-at-io-2026/
- **Pourquoi c'est important** : Positionnement agressif contre ChatGPT et Claude. Gemini Omni (vidéo IA) et Daily Brief (digest personnalisé inbox/agenda) complètent la refonte "Neural Expressive" de l'app.
- **Testable** : Oui — app Gemini (déploiement progressif)

## 🔬 Research

### MCP 2026 Release Candidate — protocole stateless
- **Quoi** : RC de la nouvelle spécification MCP disponible ; spec finale le 28 juillet 2026. Headline : MCP devient stateless au niveau protocole (HTTP ordinaire), avec extensions Tasks, MCP Apps, et durcissement OAuth/OIDC
- **Lien** : https://blog.modelcontextprotocol.io/posts/2026-mcp-roadmap/
- **Pourquoi c'est important** : La plus grande révision depuis le lancement. Le passage au stateless élimine le besoin de sessions persistantes, ce qui facilite le scaling horizontal. Cache-Control-like (`ttlMs`, `cacheScope`) sur les résultats. Les serveurs MCP peuvent désormais exposer des UIs rendues côté serveur.
- **Testable** : Oui — RC disponible sur le registry MCP officiel
