# AI Watch — 2026-06-18 Matin

## 🔥 Breaking

### Gemini CLI est mort aujourd'hui — passage forcé à Antigravity CLI
- **Quoi** : Google coupe Gemini CLI pour les comptes Google AI Pro/Ultra le 18 juin 2026 (aujourd'hui), remplacé par Antigravity CLI
- **Lien** : [Blog officiel Google Developers](https://developers.googleblog.com/an-important-update-transitioning-gemini-cli-to-antigravity-cli/) | [Guide migration](https://agentpedia.codes/blog/gemini-cli-to-antigravity-cli-migration)
- **Pourquoi c'est important** : Casse les pipelines d'automatisation existants sans parité 1:1 des features. Antigravity CLI est réécrit en Go, avec architecture async, support des subagents et plugins (anciens "Extensions"). Les licences Standard/Enterprise ne sont pas affectées.
- **Testable** : Oui — `npm install -g @google/antigravity` (ou `brew install antigravity-cli`)

### Grok V9-Medium (1.5T params) — le plus gros modèle de code d'xAI sort cette semaine
- **Quoi** : xAI finalise la mise en production de Grok V9-Medium, un modèle 1,5 trillion de paramètres (3x le modèle actuel V8), spécialement entraîné sur des workflows Cursor
- **Lien** : [TechTimes](https://www.techtimes.com/articles/318495/20260616/grok-v9-medium-arrives-spacex-seals-cursor-developers-face-model-choice-risk.htm) | [ChatForest builder guide](https://chatforest.com/builders-log/xai-grok-v9-medium-1-5t-coding-model-mid-june-2026-builder-guide/)
- **Pourquoi c'est important** : Entraîné sur les données réelles de Cursor (le workflow dev le plus utilisé), il vise directement le leadership de Claude sur les benchmarks de code. Ce n'est pas Grok 5 (6T params, à venir), mais un step intermédiaire majeur.
- **Testable** : Oui — dès sortie sur grok.com / API xAI pour SuperGrok et Premium+

## 🔬 Research

### OpenAI Deployment Simulation (16 juin)
- **Quoi** : Méthode qui rejoue des conversations passées à travers un nouveau modèle candidat avant sa release publique, pour détecter les régressions
- **Lien** : [OpenAI News](https://openai.com/news/)
- **Pourquoi c'est important** : Nouvelle approche d'évaluation pré-déploiement qui s'appuie sur des données réelles plutôt que des benchmarks synthétiques — pourrait devenir un standard industrie
- **Testable** : Non — interne OpenAI

### FrontierMath v2 — Epoch (12 juin)
- **Quoi** : Epoch publie la v2 de FrontierMath (benchmark maths avancées pour LLMs) après correction et suppression des items problématiques
- **Lien** : [Build This Now](https://www.buildthisnow.com/blog/guide/mechanics/ai-research-june-2026)
- **Pourquoi c'est important** : Benchmark de référence pour évaluer les capacités mathématiques de frontier — la v1 avait des problèmes de contamination, v2 corrige cela
- **Testable** : Non — benchmark académique

### AgentHallu — benchmark hallucinations d'agents
- **Quoi** : 693 trajectoires sur 7 frameworks d'agents et 5 domaines, pour tester si les modèles identifient quelle étape d'un pipeline multi-steps a causé une hallucination
- **Lien** : [LM Council](https://lmcouncil.ai/benchmarks)
- **Pourquoi c'est important** : Les benchmarks d'agents classiques ne testent que les outputs finaux — AgentHallu cible la cause intermédiaire, crucial pour le debug en production
- **Testable** : Non — benchmark de recherche
