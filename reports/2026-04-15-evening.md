## 🔥 Breaking

> Releases majeures, annonces qui changent la donne

### Claude Mythos Preview — Anthropic (Project Glasswing)
- **Quoi** : Modèle frontier jugé trop dangereux pour une diffusion publique — capable de détecter et d'exploiter des zero-days dans tous les OS et navigateurs majeurs
- **Lien** : [red.anthropic.com/mythos-preview](https://red.anthropic.com/2026/mythos-preview/) | [NBC News](https://www.nbcnews.com/tech/security/anthropic-project-glasswing-mythos-preview-claude-gets-limited-release-rcna267234)
- **Pourquoi c'est important** : Premier modèle public à identifier *et* weaponiser des zero-days de manière autonome. Anthropic a préféré restreindre l'accès plutôt que de le vendre. ~50 organisations (Microsoft, Nvidia, Cisco…) y ont accès via Project Glasswing avec 100 M$ de crédits pour auditer leurs propres infras. Le modèle a provoqué des réactions gouvernementales en UK.
- **Testable** : Non — accès réservé aux partenaires Project Glasswing

### GPT-5.4-Cyber — OpenAI (14 avril)
- **Quoi** : Variante cybersécurité de GPT-5.4 avec posture "cyber-permissive", reverse engineering binaire et seuil de refus abaissé pour les opérations défensives légitimes
- **Lien** : [The Hacker News](https://thehackernews.com/2026/04/openai-launches-gpt-54-cyber-with.html) | [CyberScoop](https://cyberscoop.com/openai-expands-trusted-access-for-cyber-to-thousands-for-cybersecurity/)
- **Pourquoi c'est important** : Déployé dans le cadre du programme Trusted Access for Cyber (TAC), élargi à des milliers de chercheurs individuels. Réponse directe à Claude Mythos d'Anthropic. Capacité inédite de disséquer des binaires compilés sans code source.
- **Testable** : Non — accès restreint via demande TAC (organisations vérifiées)

## 🆕 Nouveaux Outils & Produits

### Hermes Agent v0.8+ — NousResearch (open-source, MIT)
- **Quoi** : Agent autonome à mémoire persistante multi-niveaux, multi-plateformes (Telegram, WhatsApp, iMessage, WeChat, Discord, Slack, Email, CLI), avec support Android/Termux et dashboard web local
- **Lien** : [GitHub](https://github.com/nousresearch/hermes-agent) | [Site officiel](https://hermes-agent.nousresearch.com/)
- **Pourquoi c'est important** : 57 000+ stars GitHub depuis février 2026, framework agent open-source à la croissance la plus rapide de l'année. Supporte le "Fast Mode" pour OpenAI et Anthropic, et la délégation de sous-agents. La v0.8 (8 avril) ajoute les notifications en arrière-plan et le switching de modèle à chaud.
- **Testable** : Oui — `pip install hermes-agent` ou `npm install -g hermes-agent`, gratuit, open-source

### Microsoft Agent Framework 1.0 — Microsoft (3 avril)
- **Quoi** : Framework multi-agents production-ready pour .NET et Python, successeur unifié de Semantic Kernel et AutoGen, avec APIs stables et engagement LTS
- **Lien** : [devblogs.microsoft.com](https://devblogs.microsoft.com/agent-framework/microsoft-agent-framework-version-1-0/) | [Visual Studio Magazine](https://visualstudiomagazine.com/articles/2026/04/06/microsoft-ships-production-ready-agent-framework-1-0-for-net-and-python.aspx)
- **Pourquoi c'est important** : Premier framework officiel Microsoft pour agents IA multi-provider (Azure OpenAI, Claude, Gemini, Bedrock, Ollama). Support natif A2A et MCP. Pipeline middleware configurable (filtres sécurité, logs, compliance).
- **Testable** : Oui — disponible sur NuGet et PyPI, gratuit, open-source sur [GitHub microsoft/agent-framework](https://github.com/microsoft/agent-framework)

### Telnyx "LiveKit on Telnyx" (6 avril)
- **Quoi** : Plateforme hébergée pour déployer des agents vocaux IA (LiveKit) sur infrastructure Telnyx avec téléphonie enterprise et latence ultra-basse
- **Lien** : [Yahoo Finance](https://finance.yahoo.com/sectors/technology/articles/telnyx-launches-livekit-telnyx-deploying-170000808.html)
- **Pourquoi c'est important** : Supprime la complexité infra pour builder des voice agents en production. Coût réduit vs. self-hosted LiveKit + opérateur séparé.
- **Testable** : Oui — via Telnyx Portal (compte requis, usage payant)

### Voxtral TTS — Mistral AI (26 mars → disponible en avril)
- **Quoi** : Modèle TTS 4B paramètres, open weights (CC BY-NC 4.0), 9 langues, voice cloning à partir de 3s d'audio, bat ElevenLabs Flash v2.5 en naturalité
- **Lien** : [mistral.ai/news/voxtral-tts](https://mistral.ai/news/voxtral-tts) | [VentureBeat](https://venturebeat.com/orchestration/mistral-ai-just-released-a-text-to-speech-model-it-says-beats-elevenlabs-and)
- **Pourquoi c'est important** : Premier TTS compétitif open-weights qui supporte l'emotion-steering. Complète la stack Voxtral (STT + TTS). API à $0.016/1K chars pour usage commercial.
- **Testable** : Oui — modèle sur [HuggingFace mistralai/Voxtral-4B-TTS-2603](https://huggingface.co/mistralai/Voxtral-4B-TTS-2603), gratuit pour usage non-commercial

## 🧠 Mises à jour de Modèles

### Google Gemma 4 (2 avril) — Open Source, Apache 2.0
- **Quoi** : Famille de 4 modèles (2B, 4B, 26B MoE, 31B Dense), nativement multimodal (vision + audio), context 256K, 140+ langues, Apache 2.0
- **Lien** : [blog.google/gemma-4](https://blog.google/innovation-and-ai/technology/developers-tools/gemma-4/) | [HuggingFace](https://huggingface.co/blog/gemma4) | [DeepMind](https://deepmind.google/models/gemma/gemma-4/)
- **Pourquoi c'est important** : Le 31B Dense se classe #3 mondial sur Arena AI parmi les modèles open. Amélioration 20x en coding compétitif vs Gemma 3. Construit sur la même recherche que Gemini 3. 400M+ téléchargements cumulés sur les générations Gemma.
- **Testable** : Oui — HuggingFace, Kaggle, Ollama, Google AI Studio. Gratuit.

### GLM-5.1 — Zhipu AI / Z.ai (7 avril) — Open Source, MIT
- **Quoi** : 744B paramètres MoE (40B actifs/token), context 200K, MIT license — #1 mondial sur SWE-Bench Pro avec un score de 58.4 (GPT-5.4 : 57.7, Claude Opus 4.6 : 57.3)
- **Lien** : [Pandaily](https://pandaily.com/zhipu-unveils-glm-5-1-its-most-advanced-open-source-model-with-8-hour-autonomous-task-capability) | [buildfastwithai.com](https://www.buildfastwithai.com/blogs/glm-5-1-open-source-review-2026)
- **Pourquoi c'est important** : **Première fois qu'un modèle open-source dépasse tous les modèles propriétaires sur un benchmark "code repair" réel.** Entraîné sur 100 000 puces Huawei Ascend 910B (pas de GPU Nvidia) — preuve que la dépendance à Nvidia n'est pas absolue.
- **Testable** : Oui — poids sur HuggingFace (MIT), API via z.ai. Gratuit pour usage commercial.

### Meta Muse Spark (8 avril) — Propriétaire, fermé
- **Quoi** : Premier modèle propriétaire de Meta Superintelligence Labs (dirigé par Alexandr Wang post-acquisition de Scale AI pour 14 Md$)
- **Lien** : [CNBC](https://www.cnbc.com/2026/04/08/meta-debuts-first-major-ai-model-since-14-billion-deal-to-bring-in-alexandr-wang.html)
- **Pourquoi c'est important** : Marque le pivot de Meta vers des modèles fermés en parallèle de sa stratégie open-source (Llama 4). Disponible uniquement sur meta.ai. Meta espère open-sourcer de futures versions.
- **Testable** : Oui — via meta.ai (gratuit, accès web)
